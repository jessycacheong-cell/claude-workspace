What Node.js Is
JavaScript normally runs inside a browser. Node.js lets it run directly on your computer — so it can read files, run servers, and call APIs. The terminal is just how you press "go" since there's no button or screen.

What the Auth Code Is
Think of it like a ticket stub from a concert. It proves you were there (you clicked Allow). But it's not the real prize — it expires in 60 seconds and can only be used once. You have to trade it in immediately for something permanent.
⏱️ Expires in 60 seconds 🔂 Single use only 🎫 Proves you clicked Allow
5
Trade the Code for a Refresh Token
The script immediately sends the auth code back to Google along with your Client ID and Client Secret. Google verifies all three match and — if they do — sends back the Refresh Token. This is the permanent key.

What Google Checks
Auth Code — matches what it just sent? Client ID — registered app? Client Secret — correct password?
All 3 match → Google trusts the request → issues Refresh Token
⚠️
This is where we hit a bug: the POST request was missing Content-Length — the line that tells Google "here's how many bytes I'm sending." Without it, Google rejected the request with a 401 invalid_client error. Adding that one line fixed it.

Why GitHub Secrets and Not in the Code
If you put these values directly in the .yml file and pushed to GitHub, anyone on the internet could read your Gmail. Secrets are encrypted — GitHub injects them invisibly when the bot runs, and they never appear in logs or code.
7
Now It Runs Forever
The one-time handshake is done. Every 15 minutes, GitHub Actions wakes up, uses your 3 keys to get a fresh 1-hour pass from Google, reads your Gmail and Calendar, and pushes to your dashboard. No laptop. No Claude. No you.
